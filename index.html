Gestión de Sensores

Tipos de Sensores en Dispositivos Móviles

Los dispositivos móviles modernos incluyen una variedad de sensores que permiten a las aplicaciones interactuar con el entorno:

  • Acelerómetro: Mide la aceleración del dispositivo en tres ejes
  • Giroscopio: Detecta la orientación y rotación del dispositivo
  • Magnetómetro: Funciona como una brújula digital
  • Sensor de proximidad: Detecta objetos cercanos al dispositivo
  • Sensor de luz ambiental: Mide la intensidad de la luz
  • GPS: Proporciona ubicación geográfica
  • Sensor de huellas dactilares: Para autenticación biométrica

Proceso de Gestión de Sensores

La gestión de sensores implica tres pasos principales:

  1. Habilitación: Solicitar permisos y verificar disponibilidad del sensor
  2. Obtención: Configurar parámetros y comenzar a recibir datos
  3. Procesamiento: Interpretar y utilizar los datos del sensor

Figura 1: Gestion de sensores

Ejemplo de Código

En Android, puedes acceder al sensor de aceleración de la siguiente manera:

SensorManager sensorManager = (SensorManager) getSystemService(SENSOR_SERVICE);
Sensor accelerometer = sensorManager.getDefaultSensor(Sensor.TYPE_ACCELEROMETER);
sensorManager.registerListener(this, accelerometer, SensorManager.SENSOR_DELAY_NORMAL);

Servicios y Notificaciones en App

Tareas en Segundo Plano

Las aplicaciones móviles pueden ejecutar tareas en segundo plano para realizar operaciones sin interrumpir la experiencia del usuario:

  • Servicios: Componentes que ejecutan operaciones prolongadas en segundo plano
  • WorkManager: Para tareas deferibles que deben ejecutarse de manera confiable
  • AlarmManager: Para programar tareas en momentos específicos
  • JobScheduler: Para optimizar la ejecución de tareas según condiciones del sistema

Tratamiento de Notificaciones

Las notificaciones permiten a las aplicaciones comunicarse con los usuarios sin interrumpir su flujo de trabajo actual:

  1. Creación: Definir contenido, prioridad y acciones
  2. Personalización: Adaptar estilo y comportamiento
  3. Entrega: Mostrar la notificación al usuario
  4. Gestión: Manejar interacciones y actualizaciones

Depuración y Seguridad en el Desarrollo de Apps

Herramientas de Depuración

Para identificar y corregir errores en aplicaciones móviles:

  • Android Studio Debugger: Depurador integrado en el IDE oficial
  • Xcode Debugger: Para desarrollo iOS
  • Chrome DevTools: Para depurar aplicaciones web y hybrid
  • Stetho: Puente de depuración para aplicaciones Android
  • Charles Proxy: Para inspeccionar tráfico de red

Uso de Registros del Sistema

Los logs son esenciales para identificar defectos:

  1. Configuración: Definir niveles de log (VERBOSE, DEBUG, INFO, WARN, ERROR)
  2. Inserción: Agregar mensajes de log en puntos estratégicos del código
  3. Filtrado: Usar herramientas como Logcat para filtrar y buscar logs
  4. Análisis: Interpretar los logs para identificar patrones de error

Figura 1: Depuración y Seguridad

Proceso de Refactorización de Código

Mejorar la estructura del código sin cambiar su comportamiento:

  • Identificación: Detectar código duplicado, métodos largos y clases grandes
  • Planificación: Definir estrategias de refactorización
  • Ejecución: Aplicar técnicas como extracción de métodos, renombrado, etc.
  • Verificación: Ejecutar pruebas para asegurar que el comportamiento no cambió

Mecanismos de Seguridad en Apps

Protección de aplicaciones y datos del usuario:

  • Permisos: Solicitar solo los permisos necesarios y explicar su uso
  • Archivos de configuración: Proteger información sensible en build.gradle, Info.plist, etc.
  • Cifrado: Usar Android Keystore, iOS Keychain para almacenar datos sensibles
  • Autenticación: Implementar OAuth, JWT para servicios externos
  • SSL Pinning: Prevenir ataques man-in-the-middle

Herramientas para Empaquetado y Despliegue

Tipos de Empaquetado en App

Diferentes formatos para distribuir aplicaciones:

  • APK (Android Package): Formato tradicional para Android
  • AAB (Android App Bundle): Nuevo formato que permite publicación optimizada
  • IPA (iOS App Store Package): Formato para distribución en iOS
  • Formatos multiplataforma: Archivos específicos para React Native, Flutter, etc.

Herramientas y Procedimientos de Empaquetado

Proceso para preparar una aplicación para distribución:

  1. Configuración de build: Definir versionCode, versionName, y configuraciones de firma
  2. Minificación y ofuscación: Reducir tamaño y proteger código con R8/ProGuard
  3. Generación de variantes: Crear builds para diferentes entornos (debug, release)
  4. Firma: Firmar la aplicación con certificados digitales
  5. Empaquetado final: Generar APK/AAB o IPA listos para distribución

Diferenciación de Herramientas de Empaquetado

Cada plataforma tiene sus herramientas específicas:

  • iOS: Xcode Build System, Fastlane
  • Multiplataforma: Flutter CLI, React Native CLI
  • Automatización: Fastlane, Bitrise, GitHub Actions

Herramientas de Despliegue de App

Para distribuir aplicaciones a usuarios de prueba o tiendas oficiales:

Sistemas Operativos Moviles y su Arquitectura

Principales Sistemas Operativos Móviles

Los sistemas operativos móviles son plataformas que gestionan los recursos del dispositivo y permiten ejecutar aplicaciones. Los más utilizados son:

  • Android: Desarrollado por Google. Basado en Linux y de código abierto.
  • iOS: Sistema de Apple para iPhone y iPad. Enfocado en seguridad y optimización.
  • HarmonyOS: De Huawei, diseñado para múltiples dispositivos.
  • Otros: KaiOS, WearOS, sistemas embebidos especializados.

Arquitectura de Android

Android se organiza en capas que permiten modularidad y seguridad:

  1. Núcleo Linux: Controla el hardware, procesos y memoria.
  2. HAL (Hardware Abstraction Layer): Conecta el hardware con el sistema.
  3. Bibliotecas y Android Runtime (ART): Proporcionan servicios y ejecutan apps.
  4. Framework de Aplicaciones: APIs que los desarrolladores utilizan.
  5. Aplicaciones: Apps del sistema y de usuario.

Figura 1: Sistemas operativos moviles

Ejemplo de Interacción con el Sistema

En Android, obtener información del sistema operativo puede hacerse así:

String version = Build.VERSION.RELEASE;
String model = Build.MODEL;
Log.d("SO_MOVIL", "Versión: " + version + ", Modelo: " + model);

Aplicaciones Nativas y Multiplataforma

¿Qué son las Aplicaciones Nativas?

Son aplicaciones desarrolladas específicamente para un sistema operativo en particular, utilizando su lenguaje y herramientas oficiales.

  • Android: Kotlin o Java usando Android Studio.
  • iOS: Swift u Objective-C usando Xcode.

Ventajas principales:

  • Mejor rendimiento.
  • Acceso completo al hardware.
  • Mayor estabilidad.

Desventajas:

  • Mayor costo si se desarrolla para dos plataformas.
  • Código separado para cada sistema.

¿Qué son las Aplicaciones Multiplataforma?

Son aplicaciones que se desarrollan con un solo código base y funcionan en varios sistemas operativos.

Ejemplos de frameworks:

  • Flutter: Usa Dart, muy rápido y moderno.
  • React Native: Basado en JavaScript y React.
  • Ionic: Usa HTML, CSS y JavaScript.

Ventajas:

  • Un solo código para Android y iOS.
  • Menor tiempo de desarrollo.
  • Más económico.

Desventajas:

  • Rendimiento menor que nativo.
  • Algunas limitaciones al usar hardware.

Figura 1: Diferencias entre nativas y multiplataforma

Ejemplo de Comparación

// Nativo (Android - Kotlin)
val text = "Hola Android"

// Multiplataforma (Flutter - Dart)
Text("Hola desde Flutter");

Hardware de Dispositivos Móviles

¿Qué es el Hardware Móvil?

El hardware móvil es el conjunto de componentes físicos que permiten que un smartphone o tablet funcione. Determina su rendimiento, autonomía, capacidades gráficas y calidad multimedia.

Componentes principales:

  • CPU: Procesador que controla todas las tareas del sistema.
  • GPU: Encargada de los gráficos, juegos y animaciones.
  • RAM: Permite ejecutar varias apps al mismo tiempo.
  • Almacenamiento: Guarda apps, fotos y archivos.
  • Batería: Define el tiempo de uso del dispositivo.
  • Pantalla: LCD, OLED o AMOLED; resolución y tasa de refresco.
  • Sensores: Acelerómetro, giroscopio, GPS, NFC, lector de huellas, etc.
  • Cámaras: Lentes, sensores y software de procesamiento.

Arquitectura de un Dispositivo Móvil

Los móviles modernos integran varios chips en un solo módulo llamado SoC (System on Chip).

Un SoC incluye:

  • CPU (procesador)
  • GPU (gráficos)
  • IA / NPU
  • Módem de red
  • Controladores de memoria

Entornos de Desarrollo y Herramientas

Entornos de Desarrollo para Aplicaciones Móviles

Los entornos de desarrollo (IDE) permiten crear, probar y depurar aplicaciones móviles de forma eficiente.

Diseño de Interfaces de Usuario con Controles

¿Qué es el Diseño de Interfaces?

El diseño de interfaces de usuario (UI) se enfoca en cómo se presentan y organizan los elementos en una aplicación. Su objetivo es crear una experiencia clara, intuitiva y fácil de usar.

Principios básicos:

  • Claridad y legibilidad.
  • Consistencia en botones y estilos.
  • Retroalimentación visual (colores, animaciones).
  • Adaptación a diferentes pantallas.

Controles de Interfaz más comunes

Los controles son elementos interactivos que permiten al usuario realizar acciones dentro de la aplicación.

  • Botones: Ejecutan acciones.
  • Cajas de texto: Permiten ingresar información.
  • Switch o toggles: Activan/desactivan funciones.
  • Listas y RecyclerView: Muestran grupos de elementos.
  • Sliders: Ajustan valores.
  • Imágenes e íconos: Mejoran la comprensión visual.

Arquitectura y Patrones de Diseño

¿Qué es la Arquitectura de Software?

Es la organización estructural de una aplicación, definiendo cómo se divide en componentes, cómo se comunican y cómo se mantienen en el tiempo.

Importancia:

  • Facilita el mantenimiento y escalabilidad.
  • Organiza el código para evitar errores.
  • Permite trabajar en equipo de manera ordenada.
  • Mejora el rendimiento y la reutilización.

Arquitecturas más usadas en Apps Móviles

  • MVC (Model — View — Controller): Separación básica usada en frameworks tradicionales.
  • MVP (Model — View — Presenter): La vista es pasiva y el Presenter maneja la lógica.
  • MVVM (Model — View — ViewModel): Popular en Android e iOS; facilita el uso de datos reactivos.
  • Clean Architecture: Estructura el proyecto en capas independientes.
  • Bloc / Provider (Flutter): Arquitecturas reactivas basadas en streams o estados.

Figura: Ejemplo de arquitectura MVVM

Patrones de Diseño Comunes

Son soluciones probadas para problemas frecuentes en el desarrollo.

  • Singleton: Una sola instancia para recursos globales.
  • Observer: Actualiza la interfaz cuando cambian los datos.
  • Factory: Crea objetos sin exponer la lógica de creación.
  • Repository: Maneja datos de diferentes fuentes (API, BD, local).
  • Adapter: Convierte datos o interfaces para ser compatibles.

Ejemplo: Patrón Singleton (Kotlin - Android)

object ConfigManager {
  var modoOscuro = false
  fun activarModoOscuro() { modoOscuro = true }
}

Herramientas de Control de Versiones

¿Qué es el Control de Versiones?

Es un sistema que registra los cambios realizados en el código a lo largo del tiempo, permitiendo volver a versiones anteriores, colaborar con otros desarrolladores y evitar pérdida de información.

Beneficios principales:

  • Trabajo colaborativo sin conflictos.
  • Historial completo de cambios.
  • Recuperación de versiones anteriores.
  • Seguridad al trabajar en nuevas funcionalidades.
  • Facilidad para integrar código en equipo.

Herramientas de Control de Versiones más usadas

  • Git: El sistema más popular y distribuido, usado en casi todos los proyectos actuales.
  • GitHub: Plataforma para alojar repositorios Git en la nube.
  • GitLab: Plataforma con herramientas CI/CD integradas.
  • Bitbucket: Repositorios privados muy usados por empresas.

Figura: Git, la herramienta más utilizada para control de versiones

Conceptos Básicos de Git

  • Repositorio: Carpeta donde se guarda el historial del proyecto.
  • Commit: Guardado de cambios con un mensaje descriptivo.
  • Branch: Línea de trabajo para desarrollar sin afectar la principal.
  • Merge: Unir cambios de una rama a otra.
  • Clone: Descargar un repositorio remoto.
  • Push / Pull: Enviar y recibir cambios del servidor.

Ejemplo Básico de Uso de Git

git init
git add .
git commit -m "Primer commit"
git branch nueva-funcion
git checkout nueva-funcion
git push origin nueva-funcion

Persistencia y Acceso a Datos

¿Qué es la Persistencia de Datos?

La persistencia de datos es la capacidad de una aplicación para almacenar información de manera permanente, incluso después de cerrar la app o apagar el dispositivo.

Razones para usar persistencia:

  • Guardar configuraciones del usuario.
  • Almacenar datos de sesión.
  • Mantener registros, notas o listas.
  • Funcionar sin conexión a Internet (modo offline).

Tipos de Persistencia en Aplicaciones Móviles

  • Shared Preferences / UserDefaults: Datos pequeños como ajustes o banderas.
  • Archivos locales: Guardar texto, imágenes o JSON directamente.
  • Bases de datos locales: SQLite, Realm o Room (Android).
  • Almacenamiento en la nube: Firestore, MySQL remoto, Supabase, etc.
  • APIs: Acceso a datos mediante servicios web REST o GraphQL.

SQLite y Room (Android)

SQLite es el motor de base de datos integrado en Android. Room es una capa que simplifica su uso.

Ventajas:

  • Alto rendimiento.
  • No requiere Internet.
  • Ideal para apps que manejan datos estructurados.

Ejemplo básico con SQLite:

// Crear tabla
db.execSQL("CREATE TABLE usuarios(id INTEGER PRIMARY KEY, nombre TEXT)"); // Insertar datos
db.execSQL("INSERT INTO usuarios(nombre) VALUES('Juan')");

Acceso a Datos mediante APIs

Las apps modernas suelen conectarse a servidores para obtener y enviar datos mediante servicios REST o GraphQL.

Ejemplo en Android (Kotlin):

val url = "https://api.ejemplo.com/usuarios"
val request = Request.Builder().url(url).build()

Usar APIs permite:

  • Sincronizar datos entre dispositivos.
  • Guardar información en la nube.
  • Actualizaciones en tiempo real.
